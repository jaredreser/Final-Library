# Retrieval Model

The Final Library should be navigated through questions, not folders.

Users ask:

- What are the strongest untested hypotheses in this area?
- What mechanisms could explain this anomaly?
- What experiments would distinguish these theories?
- What ideas connect these two fields?
- Which claims are plausible but neglected?
- What would become possible if this hypothesis were true?
- What has already been considered and rejected?

## Retrieval Modes

### 1. Direct Search

Find records matching a claim, concept, paper, method, phenomenon, or variable.

### 2. Neighborhood Search

Find nearby hypotheses in the graph:

- Same mechanism.
- Same phenomenon.
- Same measurement.
- Same causal structure.
- Same unresolved anomaly.
- Competing explanations.

### 3. Frontier Search

Find areas with high expected value:

- High importance.
- High novelty.
- Medium-to-high plausibility.
- High testability.
- Low current evidence.
- Strong connection to unsolved problems.

### 4. Contradiction Search

Find claims that cannot all be true. This is useful for designing decisive experiments.

### 5. Experiment Search

Find hypotheses testable with a given method, dataset, instrument, population, simulation, or budget.

### 6. Translation Search

Translate machine-generated structures into human-comprehensible explanations:

- Simple explanation.
- Technical explanation.
- Mathematical form.
- Causal graph.
- Experimental protocol.
- Literature review.
- Debate map.

## Retrieval Output

A good retrieval response should include:

- A concise answer.
- Relevant hypothesis records.
- Current status of each record.
- Confidence and uncertainty.
- Evidence summary.
- Competing hypotheses.
- Suggested next tests.
- Nearby unexplored regions.

## Epistemic Labels

Every retrieval result should visibly separate:

- Established.
- Strongly supported.
- Plausible.
- Speculative.
- Generated but unreviewed.
- Contradicted.
- Falsified.
- Conceptual only.

This prevents the Final Library from becoming a hallucination engine with better filing cabinets.
